# Augment Workspace Guidelines - IMMO212 Botble Real Estate

## Project Overview

**Project Name**: IMMO212 Real Estate Platform  
**Framework**: Laravel 11 with Botble CMS  
**Environment**: SiteGround hosting with custom deployment  
**Domain**: https://staging2.immo212.ma  
**Purpose**: Real estate listings and property management platform  

## Architecture & Technology Stack

### Core Technologies
- **Backend**: Laravel 11, PHP 8.2+
- **CMS**: Botble CMS (platform-based architecture)
- **Frontend**: Blade templates, SCSS, JavaScript
- **Database**: MySQL (SiteGround hosted)
- **Asset Compilation**: Laravel Mix, npm
- **Deployment**: Custom rsync-based automation

### Directory Structure
```
├── platform/
│   ├── themes/homzen/          # Main theme (primary development area)
│   ├── plugins/                # CMS plugins
│   ├── packages/               # Core CMS packages
│   └── core/                   # CMS core functionality
├── public/themes/homzen/       # Compiled theme assets
├── resources/                  # Laravel resources
├── storage/logs/               # Error logs (monitor regularly)
└── vendor/                     # Composer dependencies
```

## Development Workflow & Preferences

### Personal Development Rules
1. **Add test scripts to gitignore** - Any temporary testing files should be automatically excluded
2. **Use existing codebase** - Prioritize extending/modifying existing code over creating new implementations
3. **Monitor Laravel error logs** - Always check `storage/logs/laravel.log` for issues
4. **Optimize deployments** - Use efficient scripts that exclude unnecessary files

### Asset Management
- **Source Files**: SCSS files in `platform/themes/homzen/assets/sass/`
- **Compiled Assets**: CSS/JS in `public/themes/homzen/`
- **Compilation**: Use `npm run build` before deployment
- **Deployment**: Only compiled assets are uploaded to server

### Theme Development
- **Primary Theme**: `homzen` located in `platform/themes/homzen/`
- **Partials**: Reusable components in `platform/themes/homzen/partials/`
- **Layouts**: Base layouts in `platform/themes/homzen/layouts/`
- **Views**: Page templates in `platform/themes/homzen/views/`

## Coding Standards & Patterns

### Botble CMS Conventions
- Follow Botble's plugin architecture for new features
- Use existing service classes and repositories
- Maintain theme structure and naming conventions
- Leverage Botble's built-in functionality before custom solutions

### Laravel Best Practices
- Use Eloquent models and relationships
- Follow PSR-4 autoloading standards
- Implement proper validation and error handling
- Use Laravel's built-in features (caching, queues, etc.)

### Frontend Development
- **SCSS Structure**: Maintain organized SCSS files with proper imports
- **Component-Based**: Use Blade components for reusable elements
- **Responsive Design**: Ensure mobile-first approach
- **Performance**: Optimize images and minimize CSS/JS

## Database & Content Management

### Database Guidelines
- **Manual Management**: Database changes are handled manually (no automated sync)
- **Migrations**: Use Laravel migrations for structural changes
- **Seeders**: Maintain seeders for development data
- **Backup**: Regular backups before major changes

### Content Structure
- **Properties**: Main real estate listings
- **Categories**: Property types and classifications
- **Locations**: Geographic organization
- **Users**: Agents, clients, administrators
- **Media**: Property images and documents
